# Smart Backup WorkflowTemplate

## Назначение

Автоматическое создание бэкапов Kubernetes ресурсов перед деплоем или другими изменениями. Сохраняет снимок состояния ресурсов, позволяя быстро восстановить их в случае проблем.

## Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `namespace` | string | `default` | Namespace, из которого делать бэкап |
| `resource-types` | string | `deployment,configmap,secret` | Типы ресурсов для бэкапа (через запятую) |
| `change-trigger` | string | `manual` | Причина бэкапа: `pre-deploy`, `manual`, `scheduled` |

## Выходные параметры

| Параметр | Описание |
|----------|----------|
| `backup-id` | Уникальный идентификатор бэкапа |
| `total-count` | Общее количество забэкапленных ресурсов |
| `backup-size` | Размер бэкапа |

## Как работает

1. Создаёт уникальный ID бэкапа с временной меткой
2. Для каждого типа ресурсов из списка:
   - Проверяет наличие ресурсов в namespace
   - Выгружает их в YAML-файлы
   - Считает количество
3. Сохраняет бэкап в ConfigMap для последующего доступа
4. Выводит сводку о созданном бэкапе

## Почему полезен

- **Безопасность**: состояние "до" всегда можно восстановить
- **Простота**: не требует внешних систем (Vault, S3)
- **Гибкость**: можно выбрать конкретные типы ресурсов и namespace
- **Прозрачность**: каждый бэкап имеет ID и временную метку
- **Автоматизация**: запускается через Argo Workflows перед деплоем

## Использование

```bash
# Ручной запуск
argo submit --from workflowtemplate/smart-backup \
  -n argo-workflows \
  -p namespace=production \
  -p resource-types=deployment,configmap,service

# Получить бэкап
kubectl get configmap backup-default-20260424-132303 -n argo-workflows -o jsonpath='{.binaryData.backup\.tar\.gz}' | base64 --decode > backup.tar.gz

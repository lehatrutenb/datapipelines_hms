## Опишите, на какие топики вы разобьёте потоки данных (законы, судебные решения, редакции документов и т.д.), сколько партиций сделаете и какие ключи будете использовать

Я выберу топики:
- raw.laws - пришедшие как есть законы
- raw.editions  - пришедшие как есть редакции
- raw.court - пришедшие как есть судебные решения
- preprocessed.court - вероятно при публикации могут упустить часть анонимизации и нужно добавить например коннектор который бы сюда перекладывал с заменой по всяким regexp, если как-то упустили это в пайплайне ранее
- preprocessed.editions - я предполагаю что внутри полноценный документы - значит последней версии будет достаточно - будем тут поддерживать сжатый вариант

Я не знаю нагрузки консультанта - пусть было принято использовать x > 1 узлов

Партиции:

- raw.laws - 2*x - вероятно крайне редкие изменения, но вот обращений прилично
- raw.editions  - 2*x - вероятно постоянные, но не достаточно частые изменения, обращения в основном архивные/из коннекторов - их просто прилично
- raw.court - 3*x много изменений, обращения в основном из коннекторов
- preprocessed.court - 6*x изменения из коннекторов и обращений
- preprocessed.editions - 6*x изменения из коннекторов, но вот обращений уже много

Ключи:

- raw.laws - строка номера закона
- raw.editions  - строка номер изменяемого документа | строка номера изменения
- raw.court - вид суда | уникальная для него идентифицакия - но функция разбиения на партиции - вид суда
- preprocessed.court - вид суда | уникальная для него идентифицакия - но функция разбиения на партиции - вид суда
- preprocessed.editions - строка номер изменяемого документа
